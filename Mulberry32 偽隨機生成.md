Mulberry32 偽隨機生成 (version 2026/3/15)

背景知識：
你知道位元運算，例如：AND、OR、XOR、NOT、左移、右移等等嗎？如果你不清楚的話，建議先去了解一下，因為這些都是偽隨機數生成器的基礎。
然後我這邊會用JavaScript來介紹，而當中會用到一些 JavaScript 的特性，例如：closure、Math.imul、>>> 0 等等，如果你不清楚的話，建議先去了解一下，這樣你就可以更好地理解這個偽隨機數生成器的運作原理了。

Mulberry32 最初是由 Tommy Ettinger 創作的一個 C 語言的偽隨機數生成器，後來被移植到 JavaScript 中，這個算法的設計目的是為了在遊戲開發中提供一個簡單、快速且具有良好統計特性的偽隨機數生成器。
來源：
https://gist.github.com/tommyettinger/46a874533244883189143505d203312c
https://github.com/cprosche/mulberry32

Mulberry32 只是 PRNG 的其中一種，PRNG 就是偽隨機數生成器（Pseudo-Random Number Generator）的縮寫。它是一種算法，可以根據一個初始值（稱為種子）來生成一系列的數字，這些數字看起來是隨機的，但實際上是完全可預測的。
但在更詳細的說明之前，我們先來看看為什麼要用偽隨機數，以及為什麼有些情況下不適合使用偽隨機數。

為什麼要用偽隨機數：
當你想要一個亂數生成，但是又想要每一個人可以得到同樣的結果。
例如你在Minecraft上在同一版本時可以生成相同的隨機地圖，那這種為隨機數生成器就很適合。

為什麼不要用偽隨機數：
因為它並不是真正的隨機數，因此用在加密的地方可能就非常不安全，因為攻擊者可以透過某些方法來預測下一個隨機數的值，或者是回推之前的隨機數的值。

而我們的 Mulberry32 會得到一個在 0 到 1 之間的浮點數，這個浮點數是根據種子值生成的，所以如果你使用相同的種子值，你就會得到相同的隨機數序列。

Mulberry32 的實現：

不囉唆，程式碼如下：

```javascript
function mulberry32(seed) {
    return function() {
        var t = seed += 0x6D2B79F5;
        t = Math.imul(t ^ (t >>> 15), t | 1);
        t ^= t + Math.imul(t ^ (t >>> 7), t | 61);
        return ((t ^ (t >>> 14)) >>> 0) / 4294967296;
    }
}
```

接下來我會解釋一下這段程式碼的運作原理，一步一步來解釋：

首先你如果有玩 Minecraft 的話，你應該知道亂數會有一個起始值，稱為種子（seed）。這個種子可以是任何整數，甚至是負數。

當你呼叫 `mulberry32(seed)` 的時候，這裡是 closure 的概念，這個函數會返回一個新的函數，這個新的函數就是我們的偽隨機數生成器。每次呼叫這個新的函數，它都會使用內部的 `seed` 來生成下一個隨機數。

假設我的起始值 seed 是 12345，我們接下來都是 uint32 的運算。

*注意：確保傳入的 seed 是整數*

那我們先來看看第一行：

```javascript
var t = seed += 0x6D2B79F5;
```

t 就是 12345 + 0x6D2B79F5，這裡的 0x6D2B79F5 是一個常數，這個常數是用來增加種子值的，這樣每次生成新的隨機數時，種子值都會改變。
t 現在是 1831578158, seed 也被更新為 1831578158

至於為什麼要用這個常數，畢竟程式設計應該盡量避免 magic number 的出現，但這個常數其實是經過精心挑選的，這樣可以確保生成的隨機數序列具有良好的統計特性，這樣就可以得到更好的隨機性了。（詳見備註 2）

接下來是這行：

```javascript
t = Math.imul(t ^ (t >>> 15), t | 1);
```
Math.imul 其實在 JavaScript 是 Integer Multiplication 的意思，而在JS的規範中其實會兩數相乘之後取模 2^32 的結果，這樣就可以確保結果在 32 位元的範圍內。
這裡的 t ^ (t >>> 15) 是將 t 右移 15 位元，然後與 t 進行 XOR 運算（先假設為 a）

則 a = t ^ (t >>> 15) = 1831578158 ^ 55895 = 1831563385

接著 t | 1 是將 t 的最低位元設為 1，這樣就確保了 t 是一個比它大的最小奇數（先假設為 b）
則 b = t | 1 = 1831578158 | 1 = 1831578159

這樣才不會因為乘上偶數而丟失最低位元的資訊，保證了映射的完整性。

接下來將 a 和 b 進行乘法運算，然後取模 2^32 的結果，這樣就得到了新的 t 的值。
t = Math.imul(a, b) = Math.imul(1831563385, 1831578159) = (1831563385 * 1831578159) | 0

但你會發現這樣數字自己在javascript驗算是對不上的，因為 Math.imul 會在 32 位元整數範圍內進行運算，超過範圍的部分會被截斷，這就是所謂的 C-style 32-bit overflow。
而且 JavaScript 的 Number 類型（IEEE 754）在超過 2^53 的整數時會失去精確度，所以在這裡我們只能得到一個近似值，這就是為什麼我們需要使用 Math.imul 來確保結果在 32 位元的範圍內。

所以你真的要驗算的話，你需要使用一個工具來模擬 32 位元整數的運算，或者是使用 BigInt 來進行計算，這樣才能得到正確的結果。（詳見備註 1）

總之 t 應該要是  Math.imul(1831563385, 1831578159) = 1291649079

接下來是這行：

```javascript
t ^= t + Math.imul(t ^ (t >>> 7), t | 61);
```
這裡的 t ^ (t >>> 7) 是將 t 右移 7 位元，然後與 t 進行 XOR 運算（先假設為 c）
則 c = t ^ (t >>> 7) = 1281686071

t | 61 則是將 t 的最低 6 位元 : 0b111101 進行 OR 運算，這樣就確保了 t 也是奇數（先假設為 d）
則 d = t | 61 = 1291649087

接下來將 c 和 d 進行 imul 運算，然後加上 t 的值，最後再與 t 進行 XOR 運算 (假設為 e)

Math.imul(c, d) = Math.imul(1281686071, 1291649087) = 1765053321
e = t + Math.imul(c, d) = 1291649079 + 1765053321 = 3056702400

接下來 將 t 與 e 進行 XOR 運算，得到新的 t 的值。
t ^ e = 1291649079 ^ 3056702400 = 4207711223
如果你算出來是 -87256073，那是因為 JavaScript 的 Number 類型在超過 2^31 - 1 的整數時會被視為負數，所以你需要將結果轉換為無符號整數，這樣才能得到正確的值。
你可以再驗算時使用 `>>> 0` 來將結果轉換為無符號整數，這樣就可以得到正確的值。

最後是這行：

```javascript
return ((t ^ (t >>> 14)) >>> 0) / 4294967296;
```

這裡的 t ^ (t >>> 14) 是將 t 右移 14 位元，然後與 t 進行 XOR 運算（先假設為 f）
則 f = t ^ (t >>> 14) = 4207711223 ^ 256818 = 4207900869 (unsigned) = -87066427 (signed)

其實這裡的 `>>> 0` 是將結果轉換為無符號整數，這樣就可以確保結果在 0 到 4294967295 的範圍內，因為 JavaScript 的 Number 類型在超過 2^31 - 1 的整數時會被視為負數，所以你需要將結果轉換為無符號整數，這樣才能得到正確的值。
所以 ((t ^ (t >>> 14)) >>> 0) 就是將 f 轉換為無符號整數，這樣就可以得到正確的值，其實就是 4207900869

最後將這個值除以 4294967296（也就是 2^32），這樣就可以得到一個在 0 到 1 之間的浮點數，這就是我們的偽隨機數了。

所以最後的結果就是 4207900869 / 4294967296 = 0.9797282677609473


呼叫的方法：

```javascript
const rand = mulberry32(12345);
console.log(rand()); // 0.9797282677609473
```
這會是第一次呼叫 rand() 的結果，如果你再次呼叫 rand()，你會得到下一個隨機數，這是因為每次呼叫 rand() 都會更新內部的 seed 值，這樣就可以生成一系列的隨機數了。
為什麼？

因為這是closure的概念，當你呼叫 `mulberry32(seed)` 的時候，它會返回一個新的函數，這個新的函數會捕獲 `seed` 這個變數，這樣每次呼叫這個新的函數時，它都可以訪問和修改 `seed` 的值，這就是 closure 的特性。所以當你呼叫 `rand()` 的時候，它會使用內部的 `seed` 值來生成下一個隨機數，然後更新 `seed` 的值，這樣就可以生成一系列的隨機數了。

如果再一次呼叫 rand()，你會得到
```javascript
console.log(rand()); // 0.3067522644996643
```

因為你是用內部新的 seed 值來生成下一個隨機數了，所以你會得到不同的結果。
是原先的 12345 + 0x6D2B79F5 的結果，1831578158。
所以第二次呼叫 時
`var t = seed += 0x6D2B79F5;`
值是被初始化為 1831578158 + 0x6D2B79F5 = 3663143971, 然後 seed 也被更新為 3663143971 讓下一次呼叫 rand() 時會使用這個新的 seed 值來生成。

然後再進行
```
t = Math.imul(t ^ (t >>> 15), t | 1);
t ^= t + Math.imul(t ^ (t >>> 7), t | 61);
return ((t ^ (t >>> 14)) >>> 0) / 4294967296;
```

如果你想要從頭開始，你就需要再次呼叫 `mulberry32(seed)`，這樣就會得到一個新的偽隨機數生成器，這個新的生成器會使用你提供的種子值來生成隨機數，這樣你就可以從頭開始了。

核心觀念就是這樣了，當然你也可以使用其他的種子值來生成不同的隨機數序列，這樣就可以得到不同的結果了。

實務上會怎麼用？

你可以用這個偽隨機數生成器來生成一些隨機的數字，例如：隨機的顏色、隨機的名字、隨機的地圖等等，這些都是在遊戲開發中常見的應用場景。

假設我想要生成一個角色的數值，例如：力量、敏捷、智力等等，我可以這樣做：

```javascript
const mrnd = mulberry32(12345);
const character = {
    strength: Math.floor(mrnd() * 100),
    agility: Math.floor(mrnd() * 100),
    intelligence: Math.floor(mrnd() * 100)
};
console.log(character);
```
應該會得到類似這樣的結果：
```javascript
{
    strength: 97,
    agility: 30,
    intelligence: 48
}
```

只要mulberry長那樣，這裏mrnd()只要是前三次，你的character的數值就會永遠是這三個數值。

## 備註 1

如果你想要在 JavaScript 中模擬 32 位元整數的運算，你可以使用 BigInt 來進行計算，這樣就可以確保結果的精確度了。以下是一個示例：

```javascript
function manualImul(a, b) {
  // 1. 使用 BigInt 確保乘法過程中低位數完全精確
  let product = BigInt(a) * BigInt(b);
  
  // 2. 取得低 32 位元 (等同於 C 語言的 uint32 溢位)
  let low32 = BigInt.asUintN(32, product);
  
  // 3. 將這 32 位元解釋為「有號整數」 (等同於 int32)
  return Number(BigInt.asIntN(32, low32));
}
```

下面這兩個值就會相等了。

```
console.log(manualImul(1831563385, 1831578159));
console.log(Math.imul(1831563385, 1831578159));
```

## 備註 2

這個常數 0x6D2B79F5 是經過精心挑選的，這樣可以確保生成的隨機數序列具有良好的統計特性，這樣就可以得到更好的隨機性了。這個常數是由 Mulberry32 的作者選擇的，根據他的測試，這個常數可以確保生成的隨機數序列具有良好的統計特性，例如：均勻分布、獨立性等等。
也可以用其他的常數來替換這個常數，但這樣可能會導致生成的隨機數序列的統計特性變差，這樣就可能會得到較差的隨機性了。

function worse_seed(seed) {
